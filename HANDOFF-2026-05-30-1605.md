**The lower-bound construction** (math is settled; this is the Lean target):
`M_K = ∏_{p≤K, prime} p^{⌊log_p K⌋+1}`, witness `n = M_K − 1`. For every `k ≤ K` and prime
`p ≤ K`: `M_K ≡ 0 mod p^{⌊log_p K⌋+1}`, so `n = M_K−1` has *maximal* base-`p` digits up through
that power ⟹ `n mod pᵃ ≥ k mod pᵃ` for all `a` ⟹ **no carries** ⟹ `vₚ(C(n,k)) = 0`. Since
`u(n,k) = ∏_{p≤k} p^{vₚ}` only involves primes `≤ k ≤ K`, all exponents vanish ⟹ `u(n,k) = 1`.
So `1 ≤ n²` always fails to be `<`, i.e. `n² < u(n,k)` is false for all `k ≤ K` ⟹ `f n > K`
(the sInf witness set excludes `[0,K]`). Then `log n ≈ log M_K = ψ(K)+θ(K) = Θ(K)` by
Chebyshev ⟹ `f n ≥ c·log n`. The `∃ᶠ` (infinitely often) comes from `K → ∞` giving distinct
`n = M_K − 1 → ∞`.

The atom that makes the `vₚ = 0` step cheap is **already proven**:
`Legendre.factorization_choose_eq_zero (hp : p.Prime) (hkn : k ≤ n) (h : ∀ i, k % p^i ≤ n % p^i)
: (n.choose k).factorization p = 0`. The work is (a) building `M_K` and proving the digit
inequality `(M_K − 1) % pᵃ ≥ k % pᵃ`, and (b) the `log M_K = Θ(K)` Chebyshev estimate.
